# Автоматизированная оценка учебных умений в МЭШ
Приложение для автоматизированной обработки журналов МЭШ / school.mos.ru с формированием рекомендаций по оценке учебных умений (ОУУ) и сохранением результатов в Excel

> Разработанная модель представляет собой детерминированную эвристико-правиловую систему педагогической диагностики, предназначенную для автоматизированного формирования рекомендаций по оценке сформированности учебных, регулятивных и коммуникативных умений обучающихся на основе анализа их цифрового следа в электронном журнале. Модель интегрирует показатели академической успешности, посещаемости, регулярности учебной активности и текстовых педагогических комментариев, преобразуя их в четырехуровневые экспертные оценки по десяти диагностическим позициям.

> Программа работает через установленный **Яндекс.Браузер** и использует профиль текущего пользователя.  
> Браузер внутрь приложения не упаковывается.

---

## Возможности

- автоматически открывает список журналов учителя в МЭШ;
- собирает журналы по всем доступным параллелям;
- извлекает данные по ученикам:
  - отметки;
  - пропуски;
  - комментарии;
  - дополнительные данные из журнала;
- рассчитывает **рекомендованные ответы** для формы «Оценить учебные умения»;
- при необходимости:
  - заполняет форму прямо в интерфейсе МЭШ;
  - автоматически сохраняет результат;
- сохраняет итог в Excel-файл:
  - лист **«Рекомендации»**;
  - лист **«Сводка»**;
- умеет пропускать уже завершённые журналы при повторном запуске.

---

## Для кого

Программа предназначена для учителя, который уже работает в **МЭШ** и хочет ускорить массовую обработку журналов.

- выполненный вход в МЭШ / mos.ru в этом браузере

Перед запуском программы **нужно полностью закрыть Яндекс.Браузер**:

2. закрыть все окна браузера;
3. проверить значок в системном трее;
4. если браузер остался в фоне — **закрыть и из трея тоже**.

## Стартовое окно

При запуске открывается окно с параметрами.

### Основные поля

#### Яндекс.Браузер (`browser.exe`)
Путь к установленному Яндекс.Браузеру.

Обычно программа пытается найти его автоматически, например в одном из стандартных путей:

- `%LOCALAPPDATA%\Yandex\YandexBrowser\Application\browser.exe`
- `C:\Program Files\Yandex\YandexBrowser\Application\browser.exe`
- `C:\Program Files (x86)\Yandex\YandexBrowser\Application\browser.exe`

Если путь определился неверно — укажите его вручную.

#### User Data
Папка пользовательских данных браузера.

По умолчанию используется:

```txt
%LOCALAPPDATA%\Yandex\YandexBrowser\User Data
```

#### Profile directory
Имя профиля внутри `User Data`.

Обычно это:

```txt
Default
```

#### Excel-файл результата
Путь к итоговому `.xlsx`-файлу.  
Если файла ещё нет — он будет создан автоматически.

  Может дать больше данных, но замедляет работу.

- **Загружать комментарии из уроков**  
  Тоже может замедлить обработку.

## Что делает программа

### 1. Открывает список журналов
Программа переходит на страницу:

```txt
https://school.mos.ru/teacher/study-process/journal/my
```

и собирает доступные журналы.

### 2. Открывает каждый журнал
Для каждого журнала программа:
- открывает таблицу;
- при необходимости включает все периоды;
- считывает данные по ученикам.

### 3. Формирует рекомендации
На основе отметок, пропусков и комментариев программа формирует рекомендованные ответы по 10 вопросам.

### 4. Заполняет форму ОУУ
Если включено заполнение в интерфейсе:
- открывается карточка ученика;
- нажимается кнопка **«Оценить учебные умения»**;
- выбираются ответы;
- при включённом автосохранении результат сохраняется.

### 5. Пишет всё в Excel
После обработки сохраняются:
- рекомендации по каждому ученику;
- сводка по каждому журналу.

По умолчанию создаётся файл:

```txt
student_skills_recommendations.xlsx
```

### Лист `Рекомендации`
Содержит данные по каждому ученику:

- параллель;
- класс;
- предмет;
- ФИО;
- отметки и комментарии;
- критерии расчёта;
- статус обработки;
- 10 рекомендованных ответов.

### Лист `Сводка`
Содержит сводную информацию по каждому журналу:

- ссылка на журнал;
- параллель;
- класс;
- предмет;
- общее количество учеников;
- сколько обработано успешно;
- сколько с неактивной кнопкой ОУУ;
- сколько с неактивной кнопкой сохранения;
- сколько с прочими ошибками.

## Значения статусов

Программа использует следующие статусы:

- **успешно сохранен**  
  Всё прошло штатно, данные сохранены.

- **кнопка ОУУ не активна**  
  Кнопка «Оценить учебные умения» была недоступна.

- **кнопка сохранения не активна**  
  Форма заполнилась, но кнопка сохранения не стала активной.

- **другая ошибка (вероятно Вы параллельно работали с журналом)**  
  Нестандартная ошибка. Частая причина — параллельная ручная работа в этом же журнале.

## Важные ограничения

### 1. Не работайте параллельно в том же журнале
Во время обработки **не рекомендуется вручную открывать и редактировать те же журналы**.

### 2. Интерфейс МЭШ может меняться
Программа ориентируется на текущую структуру страницы.  
Если МЭШ изменит интерфейс, селекторы могут перестать работать.

### 3. Рекомендации не заменяют решение учителя
Программа формирует **автоматическую рекомендацию** на основе данных журнала.  
Окончательное решение и ответственность за сохранение остаются за учителем.

### 4. Headless-режим может работать хуже
Так как используется реальный профиль браузера, обычный видимый запуск обычно стабильнее.

## Описание модели оценки учебных умений обучающегося на основе цифрового следа в электронном журнале

### 1. Общая характеристика модели

Представленная в программном коде модель относится к классу **критериально-ориентированных эвристико-правиловых моделей педагогической диагностики**. Ее назначение состоит в **автоматизированном формировании рекомендательных суждений** о степени выраженности учебных, регулятивных, познавательных и коммуникативных умений обучающегося на основе объективно фиксируемых данных электронного журнала.

С содержательной точки зрения модель не является моделью машинного обучения или статистически обучаемым предиктором. Она представляет собой **детерминированную экспертную систему**, в которой педагогически значимые выводы строятся на основе заранее заданных правил интерпретации трех групп показателей:

1. **результативных** — отметки;
2. **поведенчески-организационных** — пропуски занятий;
3. **контекстно-качественных** — текстовые комментарии учителя.

Итогом работы модели является не окончательная психологическая или педагогическая диагностика, а **рекомендация по заполнению экспертной шкалы** из 10 утверждений, отражающих частоту проявления соответствующих учебных умений. Таким образом, модель выполняет функцию **инструмента поддержки педагогического решения**, а не заменяет профессиональное суждение учителя.

---

### 2. Диагностируемый педагогический конструкт

В терминах педагогической науки модель направлена на косвенную оценку сформированности у обучающегося ряда **универсальных учебных действий и личностно-поведенческих проявлений**, а именно:

1. систематичности выполнения домашних заданий и готовности к занятиям;
2. умения выполнять задания в срок и распределять время;
3. познавательной инициативы за пределами программы;
4. интереса к задачам повышенной сложности;
5. внимательности и способности к переключению между задачами;
6. объективности самооценки результатов;
7. саморегуляции поведения и дисциплины;
8. умения слушать и поддерживать коммуникацию;
9. вежливости и соблюдения норм поведения;
10. уверенности в устном выступлении.

Следовательно, модель ориентирована не только на предметные результаты, но и на **метапредметные результаты образования**, прежде всего на **регулятивные, познавательные и коммуникативные компоненты учебной деятельности**.

---

### 3. Источники эмпирических данных

Входом модели служит агрегированный профиль обучающегося (`StudentAggregate`), в котором аккумулируются следующие данные:

- список отметок (`marks`);
- числовые значения отметок (`numeric_marks`);
- комментарии общего характера (`comments`);
- комментарии к отдельным урокам (`lesson_comments`);
- сведения о пропусках (`n_list`);
- показатель плотности отметок (`mark_density_ratios`);
- служебные сведения о классе, параллели, предмете и источнике данных.

В педагогическом смысле это означает, что модель использует **цифровой след обучающегося**, сформированный в повседневной образовательной практике.

---

### 4. Принцип операционализации признаков

#### 4.1. Количественные показатели успеваемости

Из массива отметок модель извлекает и рассчитывает следующие диагностические признаки:

- **средний балл** (`avg_mark`);
- **число отметок** (`mark_count`);
- **число низких отметок** (`low_marks`), где низкими считаются значения `≤ 2.5`;
- **число высоких отметок** (`high_marks`), где высокими считаются значения `≥ 4.0`.

При отсутствии достаточных данных используется **нейтральное значение по умолчанию**: для среднего балла — `3.7`. Это указывает на то, что модель исходит из презумпции **условно нормативного уровня**, если данных недостаточно для более точного вывода.

#### 4.2. Показатели посещаемости

Для учета пропусков используются:

- абсолютное количество пропусков (`n_count`);
- относительный показатель пропусков  
  \[
  n\_ratio=\frac{n\_count}{mark\_count+n\_count}.
  \]

Данный показатель трактуется как косвенный индикатор **включенности обучающегося в образовательный процесс**, а также его организованности и устойчивости учебной активности.

#### 4.3. Плотность учебной активности

Используется усредненный показатель `density_ratio`, отражающий **плотность выставления отметок**. В интерпретации модели этот показатель выступает как косвенная характеристика **регулярности включения ученика в оцениваемую деятельность**. Если данных нет, принимается нейтральное значение `1.0`.

#### 4.4. Устные ответы как отдельный диагностический канал

Из общего массива отметок выделяются результаты, отмеченные как **устная форма работы** (`form == "устн"`). Для них рассчитываются:

- число устных ответов (`oral_count`);
- средний балл по устным ответам (`oral_avg`).

Этот блок необходим для косвенной оценки умений, связанных с **устной речью, коммуникативной активностью и уверенностью в публичном высказывании**.

#### 4.5. Текстовые комментарии как источник качественных индикаторов

Комментарии учителя объединяются в общий текстовый массив, после чего модель ищет в нем **лексические маркеры**, связанные с определенными типами поведения. В коде реализован упрощенный механизм семантической обработки текста на основе поиска ключевых фрагментов слов.

Выделяются следующие группы маркеров:

- **опоздания** (`late_hits`);
- **использование телефона** (`phone_hits`);
- **игровое, отвлекающее, шумное поведение** (`play_hits`);
- **грубость, хамство, перебивание** (`rude_hits`);
- **неработоспособность, невыполнение заданий, неготовность** (`no_work_hits`);
- **позитивные характеристики** — «молодец», «отлично», «активен», «старается», «самостоятельный», «инициативный» (`positive_hits`).

На их основе строится обобщенный показатель поведенческих проблем:
\[
behavior\_hits = late\_hits + phone\_hits + play\_hits + rude\_hits + no\_work\_hits.
\]

С педагогической точки зрения это означает, что текстовые комментарии интерпретируются как **качественный экспертный след наблюдений учителя**, который переводится в формализованные диагностические признаки.

---

### 5. Система промежуточных диагностических индексов

Модель строит ряд промежуточных индексов, которые затем участвуют в оценке по 10 шкалам.

#### 5.1. Индекс общей учебной успешности

\[
\Delta_{avg}=
\begin{cases}
1, & \text{если } avg\_mark \ge 4.5,\\
0, & \text{если } 3.4 \le avg\_mark < 4.5,\\
-1, & \text{если } avg\_mark < 3.4.
\end{cases}
\]

Данный индекс отражает общий уровень академической результативности.

#### 5.2. Индекс плотности учебной активности

\[
\Delta_{dens}=
\begin{cases}
1, & \text{если } density\_ratio \ge 1.15,\\
-1, & \text{если } density\_ratio < 0.75,\\
0, & \text{иначе}.
\end{cases}
\]

Он интерпретируется как признак **регулярности включения в работу**.

#### 5.3. Индекс проблем посещаемости

\[
\Delta_{abs}=
\begin{cases}
-2, & \text{если } n\_ratio \ge 0.35 \text{ или } n\_count \ge 10,\\
-1, & \text{если } n\_ratio \ge 0.15 \text{ или } n\_count \ge 4,\\
0, & \text{иначе}.
\end{cases}
\]

Индекс отражает степень риска, связанного с пропусками занятий.

#### 5.4. Индекс дисциплинарной саморегуляции

\[
\Delta_{disc}=
\begin{cases}
-2, & \text{если } behavior\_hits \ge 4,\\
-1, & \text{если } behavior\_hits \ge 2,\\
1, & \text{если } behavior\_hits = 0 \text{ и } n\_ratio < 0.12,\\
0, & \text{иначе}.
\end{cases}
\]

Этот показатель характеризует выраженность признаков поведенческой организованности или дезорганизации.

#### 5.5. Индекс устной учебно-коммуникативной активности

\[
\Delta_{oral}=
\begin{cases}
1, & \text{если } oral\_count \ge 2 \text{ и } oral\_avg \ge 4.4,\\
-1, & \text{если } oral\_count \ge 2 \text{ и } oral\_avg < 3.5,\\
0, & \text{иначе}.
\end{cases}
\]

#### 5.6. Индекс устойчивости результатов

\[
\Delta_{cons}=
\begin{cases}
1, & \text{если количество отметок } \ge 4,\\
& \quad \max(numeric)-\min(numeric)\le 2.0 \text{ и } avg\_mark \ge 4.0,\\
-1, & \text{если количество отметок } \ge 4,\\
& \quad low\_marks \ge 4 \text{ и } avg\_mark < 3.3,\\
0, & \text{иначе}.
\end{cases}
\]

Этот индекс отражает стабильность учебных результатов и используется как косвенный индикатор сформированности рефлексивно-регулятивных компонентов деятельности.

### 6. Механизм формирования итоговых оценок

Каждое из 10 диагностируемых умений получает начальное значение `2`, что соответствует варианту ответа **«Часто»**. Далее это значение корректируется с учетом релевантных индикаторов. После вычисления значение ограничивается функцией:

\[
clamp(v)=\max(0,\min(3,v)),
\]

то есть все оценки приводятся к четырехуровневой шкале:

- `0` — **«Почти никогда»**;
- `1` — **«Редко»**;
- `2` — **«Часто»**;
- `3` — **«Почти всегда»**.

Таким образом, модель использует **порогово-аддитивный принцип**: к базовой оценке добавляются или вычитаются баллы, после чего результат нормируется в допустимом диапазоне.

### 7. Правила вывода по отдельным умениям

Ниже приводится содержательная интерпретация правил, заложенных в коде.

#### 7.1. Систематичность выполнения домашних заданий и готовность к урокам (Q1)

\[
Q1 = clamp(2 + \Delta_{avg} + \Delta_{dens} + \Delta_{abs} - I(late\_hits \ge 2))
\]

Показатель зависит от:
- общего качества успеваемости;
- регулярности учебной активности;
- количества пропусков;
- фактов опозданий.

Следовательно, умение трактуется как функция **учебной дисциплины и систематичности участия в работе**.

#### 7.2. Выполнение заданий в срок и распределение времени (Q2)

\[
Q2 = clamp(2 + \Delta_{dens} + I(avg\_mark \ge 4.3) - I(late\_hits \ge 1) - I(n\_count \ge 4))
\]

В основе лежат:
- регулярность работы;
- высокий средний результат;
- отсутствие опозданий и значимых пропусков.

#### 7.3. Интерес к знаниям за пределами программы (Q3)

\[
Q3 = clamp(2 + I(avg\_mark \ge 4.6 \land density\_ratio \ge 1.0) + I(positive\_hits \ge 2) - I(avg\_mark < 3.2))
\]

Операционализация строится через сочетание:
- очень высокой успеваемости;
- включенности в учебную работу;
- положительных качественных характеристик в комментариях.

#### 7.4. Интерес к более сложным задачам, любознательность (Q4)

\[
Q4 = clamp(2 + I(avg\_mark \ge 4.4) + I(density\_ratio \ge 1.15) - I(avg\_mark < 3.0))
\]

Это умение связывается с высокими и устойчивыми учебными достижениями и активностью.

#### 7.5. Внимательность и переключаемость между задачами (Q5)

\[
Q5 = clamp(2 + \Delta_{avg} - I(n\_count \ge 4) - I(behavior\_hits \ge 2) + I(high\_marks > 2\cdot low\_marks \land avg\_mark \ge 4.0))
\]

Умение интерпретируется через:
- качество учебной работы;
- отсутствие пропусков;
- отсутствие поведенческих затруднений;
- преобладание высоких результатов над низкими.

#### 7.6. Объективность самооценки (Q6)

\[
Q6 = clamp(2 + \Delta_{cons} + I(avg\_mark \ge 4.3 \land low\_marks = 0) - I(low\_marks \ge 4 \land avg\_mark < 3.3))
\]

Здесь код использует косвенную логику: объективность самооценки выводится из **стабильности и непротиворечивости результатов**.

#### 7.7. Саморегуляция поведения и дисциплина (Q7)

\[
Q7 = clamp(2 + \Delta_{disc} - I(n\_count \ge 6))
\]

Это один из наиболее прямо диагностируемых показателей, так как он связан с:
- поведением по комментариям;
- посещаемостью.

#### 7.8. Умение слушать собеседника и поддерживать разговор (Q8)

\[
Q8 = clamp(2 + \Delta_{oral} - I(behavior\_hits \ge 3) + I(positive\_hits \ge 1))
\]

Показатель строится на сочетании:
- успешности устных ответов;
- отсутствия поведенческих нарушений;
- наличия позитивных характеристик.

#### 7.9. Вежливое общение и соблюдение норм поведения (Q9)

\[
Q9 = clamp(2 + I(behavior\_hits = 0) - I(phone\_hits + play\_hits \ge 2) - I(rude\_hits \ge 1))
\]

Это умение преимущественно основывается на **качественных поведенческих маркерах**.

#### 7.10. Уверенность в выступлении перед аудиторией (Q10)

\[
Q10 = clamp(2 + \Delta_{oral} + I(oral\_count \ge 3 \land oral\_avg \ge 4.6) - I(oral\_count \ge 2 \land oral\_avg < 3.2))
\]

Диагностика строится на показателях устной активности и ее успешности.

### 8. Корректирующий механизм «позитивного порога»

В модели предусмотрен дополнительный этап, предотвращающий необоснованное занижение оценок у объективно сильных обучающихся. Если:

- средний балл `avg_mark ≥ 4.25`,
- отсутствуют низкие отметки (`low_marks = 0`),

то применяется **повышающий нижний предел** итоговых значений.

Если при этом:
- `behavior_hits < 2`,
- `n_ratio < 0.20`,

то для ответов задаются минимальные уровни:
\[
[3,3,3,3,3,3,2,3,2,3].
\]

Это означает, что по большинству показателей сильный и дисциплинированный ученик не может получить оценку ниже уровня **«Почти всегда»** или **«Часто»**. Если же высокая успеваемость сочетается с отдельными поведенческими рисками, модель устанавливает более мягкий нижний предел — не ниже уровня `2` («Часто»).

В педагогическом плане данный механизм отражает принцип: **устойчиво высокая и бездефицитная учебная результативность рассматривается как существенный аргумент в пользу высокой сформированности ряда учебных умений**, если нет выраженных контрпризнаков.

### 9. Формат результата и принцип интерпретируемости

На выходе модель формирует:

1. **список рекомендованных ответов** по 10 утверждениям;
2. **набор объясняющих критериев** (`criteria`), включающий:
   - средний балл,
   - число отметок,
   - число пропусков,
   - долю пропусков,
   - плотность отметок,
   - показатели устных ответов,
   - число низких и высоких отметок,
   - число найденных текстовых маркеров,
   - итоговые ответы по каждому пункту.

Тем самым модель обладает свойством **интерпретируемости**: педагог может увидеть, какие именно признаки повлияли на рекомендацию. В коде прямо зафиксировано, что итоговое решение остается за учителем:  
**«Рекомендация сформирована автоматически по объективным данным журнала. Итоговое решение и сохранение — за учителем».**

### 10. Научно-педагогическая интерпретация модели

С позиции педагогической методологии данную модель можно определить как:

> **цифровую критериально-уровневую модель косвенной педагогической диагностики учебных умений обучающегося, основанную на интерпретации цифрового следа его образовательной активности в электронном журнале посредством системы детерминированных экспертных правил.**

Ее принципиальные особенности состоят в следующем:

- она опирается на **объективируемые данные повседневного образовательного процесса**;
- обеспечивает **операционализацию труднонаблюдаемых метапредметных качеств** через систему косвенных признаков;
- сочетает **количественные показатели** (отметки, пропуски, плотность оценивания) и **качественные наблюдения** (комментарии педагога);
- дает **не абсолютный диагноз**, а **рекомендательное заключение**;
- сохраняет ведущую роль **экспертной педагогической интерпретации**.
